## Key Metrics Explained

### Overview Tab
```
Total People = All individuals in tree
Generations = Number of ancestral/descendant levels
Average Age = Mean age at death (or current age)
Oldest/Youngest = Extremes by age
```

### Relationships Tab
```
Spouse Relationships = Marriages/partnerships
Parent-Child = Direct ancestor/descendant links
Total = Sum of all relationships
Ratio = Quick comparison of relationship types
```

### Gender Distribution
```
Pie Chart = Visual gender breakdown
Percentages = What % of tree is each gender
Legend = Color-coded with counts
```

### Lifespan Statistics
```
Birth/Death Dates = How many have dates recorded
Average Lifespan = Typical years lived
Year Ranges = Earliest/latest births and deaths
```

## Common Questions

**Q: Why is some data missing?**
A: Tree may not have complete information. Missing birth dates, death dates, or gender records show as 0 or "N/A".

**Q: What does "Unknown" gender mean?**
A: Person's gender wasn't recorded in the tree data.

**Q: Why are timelines limited to 20 events?**
A: Performance optimization for large trees. Scroll to see more.

**Q: How often are statistics updated?**
A: Automatically when tree data changes.

**Q: Can I edit statistics?**
A: No, statistics are read-only. Edit tree data directly to change calculations.

**Q: What if my tree is very large?**
A: Statistics calculate efficiently even with 1000+ nodes. Timeline shows first 20 events.
